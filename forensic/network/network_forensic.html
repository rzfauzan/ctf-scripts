Forensic — Network Forensic

Analisis PCAP / PCAPNG

Network forensic menganalisis traffic jaringan yang terekam dalam file PCAP/PCAPNG untuk menemukan aktivitas mencurigakan, rekonstruksi data, dan identifikasi serangan. Tools utama: Wireshark, tshark, dan Scapy.

Fondasi — Format PCAP dan struktur paket

Apa itu PCAP dan bagaimana dibaca

PCAP (Packet Capture) adalah format file yang menyimpan raw network packets. Setiap paket berisi header (waktu, panjang) dan data paket lengkap dari layer 2 sampai layer 7.

## Struktur paket (OSI Layer): Layer 2: Ethernet [src MAC | dst MAC | type] Layer 3: IP [src IP | dst IP | protocol] Layer 4: TCP/UDP [src port| dst port| flags | seq/ack] Layer 7: HTTP/DNS/FTP/... [application data] ## Contoh paket HTTP GET: Frame 42: 74 bytes on wire Ethernet: src=aa:bb:cc:dd:ee:ff, dst=11:22:33:44:55:66 IPv4: src=192.168.1.100, dst=93.184.216.34 TCP: src_port=54321, dst_port=80, flags=PSH,ACK HTTP: GET /secret.txt HTTP/1.1\r\n Host: example.com\r\n User-Agent: curl/7.68.0\r\n
Fondasi — Workflow analisis PCAP

Lima langkah sistematis untuk setiap soal PCAP

1. Overview: berapa paket total? IP mana yang paling aktif? protokol apa yang dominan?
2. Filter berdasarkan protokol: http, dns, ftp, smtp, tls — fokus satu per satu
3. Follow stream: rekonstruksi percakapan lengkap dari TCP/UDP stream
4. Extract artifacts: export file, credentials, query DNS, HTTP payload
5. Identify anomali: port tidak lazim, volume besar, pola yang aneh, encoded data
Fondasi — Wireshark display filter paling penting
http dns ftp smtp tcp.port==80 ip.addr==1.2.3.4 http.request.method=="POST" dns.qry.name contains "evil" frame contains "password" tcp.flags.syn==1 && tcp.flags.ack==0 !(arp || icmp || dns) http.response.code==200 tls.handshake.type==1 tcp.stream eq 5
Di Wireshark: Statistics → Conversations untuk lihat top talkers. Statistics → Protocol Hierarchy untuk breakdown protokol. File → Export Objects → HTTP untuk extract file.

## Python untuk decode: import base64 subdomains = ["aGVsbG8=", "d29ybGQ=", "dGhpcw==", "aXNh", "c2VjcmV0", "bWVzc2FnZQ=="] for s in subdomains: decoded = base64.b64decode(s + "==").decode() # pad jika perlu print(f"{s} → {decoded}") # Output: # aGVsbG8= → hello # d29ybGQ= → world # dGhpcw== → this # aXNh → isa # c2VjcmV0 → secret # bWVzc2FnZQ== → message
Langkah 3 — Rekonstruksi pesan lengkap

Gabungkan semua chunk secara berurutan

DNS queries terurut berdasarkan timestamp — gabungkan decoded chunk secara berurutan untuk mendapat pesan asli:

Chunk 1: "hello" Chunk 2: " world" Chunk 3: " this" Chunk 4: " isa" Chunk 5: " secret" Chunk 6: " message" Pesan lengkap: "hello world this isa secret message"
## Otomatis extract dari PCAP dengan tshark + python: # Step 1: Extract semua DNS query ke evil-c2.com tshark -r capture.pcap \ -Y 'dns.qry.name contains "evil-c2.com"' \ -T fields -e dns.qry.name \ | sort -t. -k1 > dns_queries.txt # Step 2: Extract subdomain dan decode import base64, re with open('dns_queries.txt') as f: chunks = [] for line in f: subdomain = line.split('.')[0].strip() try: pad = subdomain + '=' * (-len(subdomain) % 4) decoded = base64.b64decode(pad).decode('utf-8', errors='ignore') chunks.append(decoded) except: pass print("".join(chunks)) # Pesan yang dieksfiltasi!
Pesan terkonstruksi: "hello world this isa secret message". Di skenario nyata, ini bisa berupa credentials, dokumen rahasia, atau kunci enkripsi!

Langkah 2 — Decode HTTP Basic Authentication

Basic Auth = Base64(username:password) — sangat mudah didecode!

Langkah 5 — Beaconing jitter analysis

Cara membedakan beaconing malware vs traffic normal

## Script Python untuk detect beaconing otomatis: import pyshark, numpy as np from collections import defaultdict cap = pyshark.FileCapture('capture.pcap', display_filter='tcp') # Kumpulkan timestamp per destination IP connections = defaultdict(list) for pkt in cap: try: dst = pkt.ip.dst ts = float(pkt.sniff_timestamp) connections[dst].append(ts) except: pass # Hitung interval dan jitter untuk setiap destination for ip, timestamps in connections.items(): if len(timestamps) < 5: continue timestamps.sort() intervals = np.diff(timestamps) mean_int = np.mean(intervals) jitter = np.std(intervals) # Beaconing: interval > 30s, jitter < 5% dari mean if mean_int > 30 and jitter/mean_int < 0.05: print(f"[BEACON] {ip}: interval={mean_int:.0f}s jitter={jitter:.2f}s") # → [BEACON] 185.220.101.5: interval=300s jitter=0.12s
Tools dedicated untuk beaconing detection: RITA (Real Intelligence Threat Analytics) dan AC Hunter — keduanya menganalisis PCAP dan log untuk otomatis detect beaconing.

Ringkasan C2 Detection

1. Cari traffic ke IP eksternal tidak dikenal di port non-standard

2. Hitung interval antar koneksi → jitter rendah = beaconing

3. Decode payload (Base64) → lihat perintah C2

4. Cek C2 IP di VirusTotal / AbuseIPDB / Shodan

5. Identifikasi malware family dari pattern (endpoint, interval, encoding)

Common C2 frameworks: Cobalt Strike, Metasploit Meterpreter, Sliver, Havoc

tshark — command line Wireshark

Perintah tshark yang paling sering dipakai di forensik

#### BASIC READING #### # Baca PCAP dan tampilkan summary: tshark -r capture.pcap # Filter + tampilkan field tertentu: tshark -r capture.pcap -Y "http" -T fields \ -e frame.number -e ip.src -e ip.dst \ -e http.request.method -e http.request.uri #### STATISTICS #### # Top talkers (IP conversations): tshark -r capture.pcap -q -z conv,ip # Protocol hierarchy: tshark -r capture.pcap -q -z io,phs # DNS query summary: tshark -r capture.pcap -q -z dns,tree # HTTP request summary: tshark -r capture.pcap -q -z http,tree # All credentials found: tshark -r capture.pcap -q -z credentials #### EXTRACTION #### # Export HTTP objects: tshark -r capture.pcap --export-objects http,./http_objects/ # Export FTP data: tshark -r capture.pcap --export-objects ftp-data,./ftp_objects/ # Follow TCP stream dan save: tshark -r capture.pcap -z follow,tcp,ascii,0 #### SEARCH #### # Cari string di seluruh PCAP: tshark -r capture.pcap -Y 'frame contains "password"' # Cari semua URL yang mengandung "admin": tshark -r capture.pcap -Y 'http.request.uri contains "admin"' \ -T fields -e http.request.uri # Extract semua DNS query: tshark -r capture.pcap -Y dns -T fields \ -e dns.qry.name | sort | uniq # Extract semua IP unik: tshark -r capture.pcap -T fields -e ip.src -e ip.dst \ | tr '\t' '\n' | sort | uniq #### SPECIFIC PROTOCOLS #### # FTP credentials: tshark -r capture.pcap -Y ftp -T fields \ -e ftp.request.command -e ftp.request.arg | grep -E "USER|PASS" # HTTP Basic Auth: tshark -r capture.pcap -Y "http.authorization" \ -T fields -e http.authorization # ICMP (ping-based C2 detection): tshark -r capture.pcap -Y icmp -T fields \ -e ip.src -e ip.dst -e icmp.type -e data.data #### ADVANCED #### # Deteksi beaconing (hitung interval): tshark -r capture.pcap -Y "ip.dst==185.220.101.5" \ -T fields -e frame.time_epoch \ | awk 'NR>1{print $1-prev}{prev=$1}' # Convert PCAP to JSON for further processing: tshark -r capture.pcap -T json > packets.json # Capture live traffic: tshark -i eth0 -w output.pcap
