Soal — Twin Prime Attack (Fermat Factorization)

Pecahkan RSA dengan faktor prima yang berdekatan!

Eve mencuri kunci publik dan ciphertext berikut. Diketahui bahwa p dan q yang dipakai sangat berdekatan (twin prime):

ParameterNilai
N (modulus)221
e (exponent publik)5
C (ciphertext)22

Gunakan Fermat Factorization untuk faktorkan N, lalu dekripsi C dan temukan M!

1 / 7
Langkah 1 — Verifikasi syarat serangan

Kenapa Twin Prime Attack bisa dilakukan?

Serangan ini memanfaatkan kelemahan ketika p dan q dipilih terlalu berdekatan. Identitas kuncinya:

N = p × q = (a+b)(a-b) = a² - b²

Dimana:

a = (p+q)/2 → nilai tengah antara p dan q b = (p-q)/2 → selisih setengah p dan q

Jika p ≈ q (sangat dekat), maka:

b = (p-q)/2 sangat kecil → a ≈ √N
Artinya: kita cukup coba a mulai dari ⌈√N⌉, dan dalam beberapa iterasi b² akan menjadi kuadrat sempurna!
Syarat serangan: p dan q berdekatan. Makin dekat, makin sedikit iterasi yang dibutuhkan.

Cek apakah N = 221 layak diserang:

√221 ≈ 14.87 → a mulai dari 15 Jika ketemu, berarti p dan q memang berdekatan ✓
Langkah 2 — Fermat Factorization (iterasi)

Mulai dari a = ⌈√N⌉ = ⌈√221⌉ = 15

Setiap iterasi: hitung b² = a² - N, cek apakah b² adalah kuadrat sempurna (b = integer).

a b² = a² - N b = √b² Status
15 15² - 221 = 225 - 221 = 4 √4 = 2 Ketemu!
Hanya 1 iterasi! b² = 4 adalah kuadrat sempurna → b = 2 (integer).
Kalau b² bukan kuadrat sempurna (hasilnya desimal), lanjut ke a+1 dan ulangi. Contoh: kalau N=15, a=4 → b²=1, b=1 ✓ langsung dapat juga.
a = 15, b = 2 → siap faktorkan N!
Langkah 3 — Hitung p dan q

Gunakan a dan b untuk temukan faktor prima

Dari identitas N = (a+b)(a-b), maka:

p = a + b q = a - b p = 15 + 2 = 17 q = 15 - 2 = 13 p = 17, q = 13

Verifikasi faktorisasi:

p × q = 17 × 13 = 221 = N ✓
N berhasil difaktorkan! Dengan p dan q diketahui, RSA sudah runtuh.

Kenapa ini bahaya? Karena kunci privat RSA dihitung dari φ(N) = (p-1)(q-1), dan sekarang kita tahu p dan q-nya.

Langkah 4 — Hitung φ(N)

φ(N) = (p-1)(q-1)

φ(N) atau Euler's totient dibutuhkan untuk menghitung kunci privat d.

φ(N) = (p - 1) × (q - 1) φ(221) = (17 - 1) × (13 - 1) = 16 × 12 φ(N) = 192
φ(N) = jumlah bilangan bulat 1..N yang coprime dengan N. Untuk RSA semiprime, rumusnya selalu (p-1)(q-1).

Selanjutnya kita pakai φ(N) = 192 untuk hitung kunci privat d.

Langkah 5 — Hitung kunci privat d

d = e⁻¹ mod φ(N) = 5⁻¹ mod 192

Cari d sehingga e × d ≡ 1 (mod φ(N)), yaitu 5d ≡ 1 (mod 192). Gunakan Extended Euclidean:

Fase maju:

192 = 38 × 5 + 2 5 = 2 × 2 + 1 2 = 2 × 1 + 0 ← gcd = 1

Fase mundur (substitusi balik):

1 = 5 - 2 × 2 = 5 - 2 × (192 - 38 × 5) = 5 - 2×192 + 76×5 = 77×5 - 2×192 5⁻¹ ≡ 77 (mod 192) → d = 77

Verifikasi:

e × d = 5 × 77 = 385 385 mod 192 = 385 - 2×192 = 385 - 384 = 1 ✓
d = 77 adalah kunci privat yang berhasil kita rekonstruksi!
Langkah 6 — Dekripsi C menggunakan d

M = C^d mod N = 22^77 mod 221

Hitung 22^77 mod 221 menggunakan fast modular exponentiation (eksponen dalam bentuk biner).

77 dalam biner = 64 + 8 + 4 + 1 = 1001101₂

Langkah 7 — Verifikasi

Enkripsi ulang M=3 dan bandingkan

C = M^e mod N = 3^5 mod 221 3^5 = 243 243 mod 221 = 243 - 1×221 = 22 = C ✓
Ciphertext cocok! Serangan berhasil.
Plaintext ditemukan
M = 3

Ringkasan langkah Twin Prime Attack

1. Identifikasi: p ≈ q → Fermat Factorization

2. Hitung a = ⌈√N⌉, iterasi sampai b² = a²-N adalah kuadrat sempurna

3. Dapat p = a+b, q = a-b

4. Hitung φ(N) = (p-1)(q-1)

5. Hitung d = e⁻¹ mod φ(N) (Extended Euclidean)

6. Dekripsi M = C^d mod N (fast modular exponentiation)

7. Verifikasi: C = M^e mod N
