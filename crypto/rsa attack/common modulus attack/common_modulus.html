Soal — Common Modulus Attack

Temukan plaintext M!

Dua pengguna (Alice dan Bob) menggunakan modulus N yang sama tetapi exponent publik berbeda. Eve menyadap kedua ciphertext dari pesan M yang sama:

PenggunaModulus NExp. Publik (e)Ciphertext (C)
Alice55e₁ = 3C₁ = 8
Bob55e₂ = 7C₂ = 18

N sama = 55  |  C₁ = M^e₁ mod N  |  C₂ = M^e₂ mod N  |  Cari M!

1 / 6
Langkah 1 — Verifikasi syarat serangan

Cek semua syarat Common Modulus Attack

Ada 3 syarat yang harus terpenuhi:

Syarat 1: N sama untuk kedua penerima → N = 55 ✓
Syarat 2: Pesan M yang sama dienkripsi ke keduanya ✓

Syarat 3: gcd(e₁, e₂) = 1 → wajib coprime!

gcd(e₁, e₂) = gcd(3, 7) = ?

3 prima, 7 prima, keduanya berbeda → tidak ada faktor persekutuan

gcd(3, 7) = 1 ✓
Semua syarat terpenuhi! Serangan bisa dilanjutkan.
Kenapa gcd harus = 1? Karena kita butuh r dan s sehingga r·e₁ + s·e₂ = 1. Ini hanya mungkin jika gcd(e₁,e₂) = 1 (teorema Bezout).
Langkah 2 — Ide utama serangan

Kenapa serangan ini bisa bekerja?

Kita punya dua persamaan:

C₁ = M^e₁ mod N → 8 = M³ mod 55 C₂ = M^e₂ mod N → 18 = M⁷ mod 55

Karena gcd(e₁, e₂) = 1, ada bilangan bulat r dan s dimana:

r·e₁ + s·e₂ = 1

Maka:

C₁ʳ · C₂ˢ mod N = (M^e₁)ʳ · (M^e₂)ˢ mod N = M^(r·e₁) · M^(s·e₂) mod N = M^(r·e₁ + s·e₂) mod N = M^1 mod N = M
Tanpa tahu kunci privat, kita bisa dapat M langsung dari C₁ dan C₂!
Kunci langkah berikutnya: cari r dan s menggunakan Extended Euclidean Algorithm pada e₁ dan e₂.
Langkah 3 — Extended Euclidean Algorithm

Cari r, s sehingga r·e₁ + s·e₂ = 1   (3r + 7s = 1)

Jalankan Extended Euclidean pada pasangan (e₁=3, e₂=7). Kita cari dari yang lebih besar:

Fase maju (pembagian berulang):

7 = 2 × 3 + 1 3 = 3 × 1 + 0 ← berhenti, gcd = 1

Fase mundur (substitusi balik):

Dari baris pertama: 1 = 7 - 2×3 Tulis dalam bentuk r·e₁ + s·e₂: 1 = (-2)×3 + (1)×7 r = -2, s = 1

Verifikasi:

r·e₁ + s·e₂ = (-2)×3 + (1)×7 = -6 + 7 = 1 ✓
r = -2 (NEGATIF)! Ini masalah, karena C₁^(-2) tidak langsung bisa dihitung secara modular. Langkah berikutnya menangani ini.
Langkah 4 — Tangani eksponen negatif

r negatif → gunakan invers modular C₁

Karena r = -2, kita tidak bisa langsung hitung C₁^(-2). Tapi secara modular:

C₁^(-2) mod N = (C₁⁻¹ mod N)² mod N

Jadi kita perlu hitung C₁⁻¹ mod N = 8⁻¹ mod 55 dulu.

Extended Euclidean untuk 8⁻¹ mod 55:

55 = 6×8 + 7 8 = 1×7 + 1 7 = 7×1 + 0 ← gcd = 1

Substitusi balik:

1 = 8 - 1×7 = 8 - 1×(55 - 6×8) = 8 - 55 + 6×8 = 7×8 - 1×55 8⁻¹ ≡ 7 (mod 55)

Verifikasi:

8 × 7 = 56 = 1×55 + 1 → 56 mod 55 = 1 ✓
C₁⁻¹ mod 55 = 7. Sekarang C₁^(-2) = 7² = 49.
Langkah 5 — Hitung M

M = C₁ʳ · C₂ˢ mod N

Semua komponen sudah siap. Masukkan r=-2, s=1, C₁=8, C₂=18, N=55:

KomponenNilaiCara hitung
r-2dari Extended Euclidean
s1dari Extended Euclidean
C₁^r = C₁^(-2)(C₁⁻¹)² = 7² = 49karena r negatif
C₂^s = C₂^118langsung
M = C₁^r · C₂^s mod N M = 49 · 18 mod 55 49 × 18 = 882 882 ÷ 55 = 16 sisa 2 (16 × 55 = 880) 882 - 880 = 2 M = 2
Langkah 6 — Verifikasi

Enkripsi ulang M=2 dan bandingkan

C₁ = M^e₁ mod N = 2³ mod 55 = 8 mod 55 = 8 ✓ C₂ = M^e₂ mod N = 2⁷ mod 55 = 128 mod 55 128 = 2×55 + 18 → C₂ = 18 ✓
Kedua ciphertext cocok! Serangan berhasil.
Plaintext ditemukan
M = 2

Ringkasan langkah Common Modulus Attack

1. Verifikasi: N sama, gcd(e₁,e₂) = 1

2. Pahami ide: r·e₁ + s·e₂ = 1 → M = C₁ʳ·C₂ˢ mod N

3. Extended Euclidean pada (e₁, e₂) → dapat r dan s

4. Jika r atau s negatif → hitung invers modular Cᵢ⁻¹ mod N

5. Hitung M = C₁ʳ · C₂ˢ mod N

6. Verifikasi dengan enkripsi ulang
