Fase 1 — Recon

Kenali perilaku oracle

Sebelum menyerang, kita harus pahami dulu: seberapa panjang output oracle bertambah kalau kita tambah input? Ini memberi tahu ukuran blok dan panjang secret.

Coba kirim string berulang dengan panjang berbeda dan catat kapan output bertambah 16 byte:
Perhatikan: output bertambah 16 byte (32 hex chars) setiap kali input melewati kelipatan 16. Ini konfirmasi block size = 16 byte. Panjang SECRET bisa dihitung dari kapan output pertama kali bertambah.
Fase 2 — Block Alignment

Isolasi satu blok untuk diserang

Secret = 8 byte. Idenya: buat input sedemikian rupa sehingga satu blok berisi (15 byte inputmu) + (1 byte pertama secret). Lalu kita bisa brute-force byte itu.

Struktur enkripsi oracle:
[ input_kamu (n byte) | SECRET (8 byte) ]

Strategi: kirim 15 "A" → blok pertama jadi:
[ A A A A A A A A A A A A A A A S₁ ]
← 15 A kita → ← 1 byte secret → = 16 byte = 1 blok penuh

Blok pertama yang akan kita jadikan target:
Ini adalah enkripsi dari [15×A + byte_pertama_secret]. Kita akan cocokkan ini!
Kita perlu tahu: AES_K("AAAAAAAAAAAAAAA" + X) = apa, untuk setiap kemungkinan X (32 kemungkinan karakter printable yang mungkin). Kita bandingkan hasilnya dengan target block di atas.
Fase 3 — Byte-by-Byte Extraction

Brute-force secret byte per byte

Untuk tiap byte secret, kita:

1. Kirim (15 - posisi) "A" → isolasi 1 byte secret di ujung blok
2. Brute-force: kirim (15 - posisi) "A" + karakter candidate untuk semua 95 printable chars
3. Bandingkan blok pertama → yang cocok = byte secret!
4. Ulangi untuk posisi berikutnya
